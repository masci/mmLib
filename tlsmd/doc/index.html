TLS Motion Determination (TLSMD) analyzes a protein crystal structure for evidence of flexibility, in the form of local or inter-domain motions. It does this by partitioning the protein structure into multiple segments which are modeled as rigid bodies undergoing TLS (Translation/Libration/Screw) vibrational motion. It generates all possible segmentations up to a specified maximum number of TLS groups. Each trial segmentation is evaluated by comparison to the observed atomic displacement parameters (ADPs, "thermal parameters") that came out of crystallographic refinement.

TLSMD generates the optimal 1-group TLS model (trivial), the optimal 2-group TLS model, and so on up to a default of 20 TLS groups total. For each such model you get statistics describing how well this model accounts for the thermal motion observed in the crystal structure.

You also get:
  • Various plots and graphical images showing the implied inter-domain or other motions.
  • An animation of the implied motion via the Jmol Java applet
  • Modified PDB files and TLS input files that can be used for further crystallographic refinement using the CCP4 program Refmac5
  • These same files can be used for more detailed inspection in the Python Macromolecular Library's TLSView interactive viewer, which offers a wide range of static and animated representations of the implied motions.

Submit your structure by visiting the Start New Job page and completing the 3-step submission process. TLSMD is computationally expensive, so it uses a batch processing system to run submitted structures one at a time. You may monitor the progress of your submitted jobs through the Job Status page.

More information about TLS groups can be found in the TLSView Manual. It covers interpretation of the TLS parameters, and the structure visualizations generated by the TLSMD analysis. TLSView is part of the Python Macromolecular Library (mmLib), and is specifically designed to be a interactive molecular viewer of the TLS groups generated by TLSMD.
