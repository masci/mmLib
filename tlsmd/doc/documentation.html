Introduction

TLSMD analyzes a protein crystal structure for evidence of flexibility, in the form of local or inter-domain motions. It does this by partitioning the protein structure into multiple segments which are modeled as rigid bodies undergoing TLS (Translation/Libration/Screw) vibrational motion. It generates all possible segmentations up to a specified maximum number of TLS groups. Each trial segmentation is evaluated by comparison to the observed atomic displacement parameters (ADPs, "thermal parameters") that came out of crystallographic refinement.

TLSMD generates the optimal 1-group TLS model, the optimal 2-group TLS model, and so on up to a default of 20 TLS groups total. For each such model you get statistics describing how well this model accounts for the thermal motion observed in the crystal structure.

More Information about TLS groups can be found in the TLSView Manual. It covers interpretation of the TLS parameters, and the structure visualizations generated by the TLSMD analysis. TLSView is part of the Python Macromolecular Library (mmLib), and is specifically designed to be a interactive molecular viewer of the TLS groups generated by TLSMD.

You also get:
  • Various plots and graphical images showing the implied inter-domain or other motions.
  • An animation of the implied motion via a Jmol javascript applet
  • Modified PDB files and TLS input files that can be used for further crystallographic refinement in the CCP4 program Refmac5
  • These same files can be used for more detailed inspection in the Python Macromolecuar Library's TLSView interactive viewer, which offers a wide range of static and animated representations of the implied motions.

Note: This is a beta-test version of TLSMD. We expect it to change rapidly. Please report any problems, suggestions, or blinding revelations inspired by the analysis. If you publish an analysis based on the TLSMD results, please cite Painter & Merritt (2005) Acta Cryst. D61, 465-471.

Structure Submission

User Information
Fill out the information below to complete the submission of your structure to TLSMD. If you would like to password protect your submission, then make sure to fill out the user and password fields, otherwise your submission will be publicly accessible. Entering your email address is optional, we use it only as a contact address if there is a problem with processing your submission.

Processing Time
TLSMD runs a independent analysis on each protein chain present in your input PDB file. The time required to analyze each chain is roughly proportional to the square of the number of residues in the chain, and the total analysis time for your structure is the sum of the time taken for each chain. On our current computer, a single 300 residue protein chain takes about 2 hours to analyze; a homo-dimer with two such chains would take 4 hours. For this reason, we ask users submitting structures with protein chains larger than 600 residues to use the Include Atoms: Mainchain Only option on the submission form below to decrease the analysis time required for your protein.

Cancelling Your Submitted Job
If there is a problem with your submission, you may edit or kill the submission through the Edit Job option on the TLSMD web site. Once submitted, you may monitor its status on the Job Queue, and once it is complete it will be moved into the Completed Jobs list. Both of these web pages are accessible from the TLSMD main website.

Submission Options
There are three options controlling the minimization of your structure: TLS Model, Least Squares Weighting, and Include Atoms.

TLS Model
The TLS Model option selects either the isotropic form of the TLS equations, or the anisotropic form. If your structure has been refined with either isotropic atomic displacement parameters (ADPs)(the parameter formerly known as temperature factor), or TLS refinement, then select the isotropic model which is the default. If you are lucky enough to have 1.0 Angstrom data and used anisotropic ADPs refinement and you have ANISOU records in your PDB file, then select the anisotropic TLS model.

Least Squares Weighting
The TLS model parameters are fit to the individual atomic ADPs of your input model using a overdetermined system of linear equations. In this system of equations, each atom can be given a unique weight. Usually, the weight is the inverse of the standard deviation of the parameter (in this case the standard deviation of the atomic ADP in your input model); however, the standard deviation of ADPs are usually not known. It is reasonable to assume the ADP errors are inversely proportional to their Biso values, so this is a optional least squares weighting choce for the atom in TLSMD.

Include Atoms
This option controls the subset of atoms used when fitting the TLS model to your refined structure. If you have chains with more than 600 residues, then please only use the main chain atoms, otherwise your job could take a long time to run.
